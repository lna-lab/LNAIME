# LNAIME（リナイム）

**by Lna-Lab** — ケンとユキの、**完全ローカル・出版編集者グレードの日本語入力＆校正IME**。
名称 = **L**na-Lab + **IME**。ローマ字ライブ変換（ニューラル）＋出版グレード校正を、**原稿が箱の外に一歩も出ない**形で。

> **形：常駐IMEファースト**（ケン決定 2026-06-09）。Google日本語入力 / ATOK / 英かな のように、
> **入力デバイスの近くに常駐し、どのアプリでも効く**。専用エディタは作らない。

## なに / なぜ
- 対象＝プロの出版編集者。執筆・編集の常用IME、リリース品質。
- 差別化＝**完全ローカル**（クラウド校正＝文賢/Grammarlyに対する信頼）＋**君の固有辞書**＋ニューラル変換。
- 起点＝X発「ローマ字のままAIに投げる」。①人→AIは変換不要、②人→人/文書は変換＋校正が要る → 本品は②。

## 校正の文書スコープ問題と、その解
ATOKが変換時に ら抜き・誤用・表記ゆれ を下線で出す——あれを**ローカルで・ニューラル変換と一緒に・君の辞書で**やる。
- **登録済みの「正」があれば、ほとんどの校正はその場で決まる**（`サーバー`が正と辞書が知っていれば、`サーバ`を打った瞬間に指摘）→ **IME層で出せる**：登録表記ゆれ／用字用語／送り仮名／ら抜き。
- IMEの鍵穴で見えないのは「**未登録語が、離れた場所で割れている**」場合だけ → **「文書スキャン」モード**（同じ脳を叩く脇役）で回収。

## 設計の芯：「1つの固有辞書を、3人が食う」
1レコード登録 → ①ライブ変換バイアス ②再変換 ③校正の正規形。一次ソースで実在確認済み（azooKey: `DicdataElement(固有名詞, value:-10)` ＋ Zenzai `辞書:` プロンプト）。

## 5層（ローカル「脳」＝1サーバ ＋ IMEフロント）
- **L0** 共有固有辞書（master 1レコード → 3成果物へコンパイル＝THE BRIDGE）
- **L1** 文字正規化（neologdn風, 決定的）
- **L2** 決定的ルール層（textlint+prh ＋ Sudachi正規化クラスタ ＋ 内閣告示 ＋ JTF）＝**再現可能・引用可能**（校正の主役）
- **L3** ニューラル層（局所LFM2.5）＝文脈依存の誤用/係り受けのみ・最小編集diff・「AI提案」と明示
- **フロント**＝Fcitx5 IME（入力＋再変換＋その場校正の下線）
- ※「正」の出所は常に L0登録正規形/告示。Sudachi正規化は**検出専用**（うどん→饂飩で暴れる）。

## THE BRIDGE（製品の核・前例なし・**稼働中**）
`compiler/compile_dict.py`：`dict/master.yaml` の1レコード → `dict/build/{prh.yml, sudachi_user.csv(11=読み/12=正規化), azookey_userdict.json}` を同期生成。

## フェーズ（IME優先・縦書きなし）
1. **脳**：L0辞書＋コンパイラ＋Sudachi/textlint/prh（固有辞書登録＆表記ゆれ校正。フレームワーク非依存）← いまここ
2. **常駐IME**：Fcitx5＋Hazkey-fork — ライブ変換✅＋固有辞書取込＋再変換＋その場校正下線（**要 IBus→Fcitx5 切替**）
3. **文書スキャン校正**：離れた未登録ゆれ＋一括校正（同じ脳）
4. ニューラルL3 ＋ 入稿(docx/PDF)　※**縦書きは不要**（ケン決定）

## 実機の現実 & first light（2026-06-09）
- Ubuntu 26.04 / RTX PRO 2000 Blackwell ×6 @16GB。CUDAビルドは **pip cu13(=13.2 nvcc) + g++-13, sm_120**（apt 12.4 と full 13.1+glibc2.41 は不可）。入力は現状 **IBus**（Fcitx5未導入）。表示=GPU3。
- **latency ゲート通過 ✅**：zenz-v3.1-small Q5_K_M、1変換 **~9–21ms**（decode 1,548 t/s = 0.65ms/tok。overhead律速＝**大型化の余地あり**）。
- zenz = gpt2系 char, vocab 6000, ctx 1024, CC-BY-SA-4.0。ライブ変換はロジットレベル＝**vLLM不可**、自前は zenz形式GGUF を訓練（当面 stock zenz-v3.1）。
